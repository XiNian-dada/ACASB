## 10. 前端对齐注意事项

### 10.1 不要假设所有枚举值都齐全

当前真实数据里：

- 朝代只有：`清 / 明 / 唐 / 宋`
- 等级只有：`官员 / 平民 / 皇家`
- 场景只有：`建筑外观 / 建筑群 / 园林 / 街景`

如果图表 UI 固定要求更多枚举项，前端应自行补零，而不要假设后端一定返回。

### 10.2 图片统一使用 `imageUrl`

不要使用：

- `storedImagePath`
- `sourceImagePath`

前端渲染图片时，只用：

- `imageUrl`

### 10.3 单条记录同时包含 AI 数据和传统特征

在替换最新 JAR 并重启服务后，`/api/dataset/*` 查询会同时返回：

- AI 标注字段
- `rawMetadata`
- `traditionalFeatures`

其中 `traditionalFeatures` 包含 19 维传统手工特征和 `royalRatio`。

### 10.4 推荐前端优先调用的接口

```text
GET /api/dataset/stats/overview?datasetName=competition-dataset
GET /api/dataset/stats/colors?datasetName=competition-dataset&limit=20
```
